# URL-Album 2 — Architectural Decisions

---

## 1. Portable mode: album.db рядом с exe

**Решение:** `std::env::current_exe()?.parent()?.join("album.db")`

**Отклонено:** `app.path().app_data_dir()` (Tauri default → AppData)

**Почему:** Оригинальный URL-Album был полностью portable: один exe + один dat
рядом. Пользователь хочет переносить папку и чтобы всё работало без
переустановки. AppData нарушает это свойство.

---

## 2. Vanilla JS, без фреймворка

**Решение:** Чистый HTML + CSS + JS, `withGlobalTauri: true`, без npm-зависимостей

**Отклонено:** Svelte, React, Vue

**Почему:** MVP должен быть стабильным и максимально простым. Без бандлера —
нет шага сборки фронтенда, нет node_modules, нет версионных конфликтов.
Tauri 2 с `withGlobalTauri` инжектирует `window.__TAURI__` — этого достаточно.

---

## 3. rusqlite с bundled feature

**Решение:** `rusqlite = { version = "0.31", features = ["bundled"] }`

**Отклонено:** системный SQLite, другие ORM (diesel, sqlx)

**Почему:** bundled = SQLite компилируется в бинарник. Нет зависимости от
системной dll. Portable. Нет ORM — нет лишних абстракций, запросы читаемы.

---

## 4. Синхронный Rust, Mutex<Connection>

**Решение:** Один `Mutex<rusqlite::Connection>` в AppState, без async

**Отклонено:** tokio + sqlx async, connection pool

**Почему:** MVP. Приложение однопользовательское, запросы мгновенные. Async
добавляет сложность без реальной выгоды. Mutex гарантирует безопасность
без лишних зависимостей.

---

## 5. SQLite вместо та-же-структура что и ua.dat (TSV)

**Решение:** SQLite с таблицей `nodes` (единая таблица для папок и закладок)

**Отклонено:** хранить данные в том же TSV-формате, что и оригинал

**Почему:** TSV хрупок при ручном редактировании, не поддерживает поиск,
нет транзакций. SQLite: один файл, быстрые запросы, атомарные записи.

---

## 6. Единая таблица nodes (kind = folder | bookmark)

**Решение:** Одна таблица `nodes` с полем `kind`

**Отклонено:** Две таблицы `folders` и `bookmarks`

**Почему:** Дерево проще строить из одной таблицы. Меньше JOIN-ов.
Глубина иерархии не ограничена. Аналог adjacency list pattern.

---

## 7. asset:// protocol для PNG, не base64

**Решение:** `tauri = { features = ["protocol-asset"] }` + `convertFileSrc()` в JS

**Отклонено:** читать PNG в Rust и возвращать base64 через IPC

**Почему:** base64 — копирование данных через IPC на каждую карточку.
asset:// — прямая загрузка файла WebView-ом, без копирования. Быстрее,
меньше памяти, особенно при большом количестве карточек.

---

## 8. encoding_rs для импорта Windows-1251

**Решение:** `encoding_rs = "0.8"` — одна функция при импорте

**Отклонено:** ручная таблица перекодировки, игнорирование кодировки

**Почему:** Все данные в ua.dat на кириллице в Windows-1251. Без декодирования
все заголовки и заметки будут мусором. encoding_rs — стандартная, маленькая,
без транзитивных зависимостей.

---

## 9. Импорт — одноразовая операция через UI

**Решение:** Экран импорта при пустой БД, ручной ввод пути к ua.dat

**Отклонено:** авто-импорт при каждом запуске, CLI-флаг, drag&drop файла

**Почему:** Простота. Данные импортируются один раз, дальше работает SQLite.
`find_uadat()` пробует автодетект рядом с exe — снижает трение.

---

## 10. open_url через std::process::Command

**Решение:** `cmd /c start "" <url>` на Windows

**Отклонено:** `tauri-plugin-shell` (открывает URL через плагин)

**Почему:** Избегаем лишней зависимости. `cmd /c start` — стандартный
Windows-способ, работает без дополнительных разрешений Tauri.

---

## Что намеренно НЕ сделано в v1

- Нет browser extension (сложность без ценности для MVP)
- Нет localhost API (то же)
- Нет авто-скриншотов (требует headless browser или скрытый WebView)
- Нет async (не нужен при синхронных локальных операциях)
- Нет тегов (иерархия из оригинала достаточна)
- Нет облака (принципиальная позиция проекта)
